## Recurring themes

### 1. Architecture sketches under-specified library semantics
Every story past 1.1 surfaced one library-or-spec mismatch. The dev had to invent or correct a seam:
- 1.2 — `validation_alias` on nested `BaseModel` doesn't surface in `_provenance` recording → `model_validator(mode="before")` instead.
- 1.3 — `Output` protocol sketch had only `progress`/`emit`; `error()` added at dev time so the entrypoint's exception path has a redacted sink.
- 1.4 — `parsedate_to_datetime` returns naive datetime; naive `.timestamp()` interpreted in local tz. Patch routed through `datetime.timezone.utc`.
- 1.5 — Link-header parser sketch had 6 RFC-8288 defects, including an SSRF-relevant scheme-relative-URI ambiguity.
- 1.7 — `@modern_di_typer.inject` reads container from `app.info.context_settings`, set once via `setup_di(...)`; semvertag rebuilds the container per CLI invocation. Decorator abandoned; resolved providers via `container.resolve_provider(...)` inline.

**Pattern that emerged:** `Dev Agent Record §Debug Log References` became the canonical place to record "spec said X, library does Y, we did Z." By 1.5 the discipline was reliable and future stories could grep prior records.

### 2. Protocol modules need a structural-conformance unit test to count for coverage
`providers/_base.py` (1.5) and `strategies/_base.py` (1.6) both sat at 0% line coverage until a unit test imported the Protocol explicitly. Production paths never import the Protocols — DI Groups construct concrete classes directly. Pattern propagates to Epic 2's `ConventionalCommitsStrategy`.

### 3. Lint / typing tool friction
- `auto-typing-final` aggressively rewrote `yield None` → `return None` in 1.2's conftest → standing instruction to pre-annotate `typing.Final` on every module-level constant.
- `PLC0415` (no function-local imports) tripped 1.1's smoke test, then constrained 1.7's lazy-import discipline (6 `# noqa: PLC0415` in `ioc.py` — only those for the architecturally-mandated provider/strategy module boundary survived the 1.7 post-review cleanup).
- `PLR2004` magic-number forced private constants: `_RETRY_AFTER_STATUS` (1.4), `_HTTP_TOO_MANY_REQUESTS`/`_SERVER_ERROR_MIN/MAX` (1.5).
- `S607` partial-executable-path on grep test (1.4) → resolved with `shutil.which`.

### 4. CLI runner / test fixture drift
- 1.7 discovered Click ≥8.2 removed `CliRunner(mix_stderr=False)`. Spec's prescribed pattern was no longer valid; substitution accepted (Click 8.2 separates streams by default).
- 1.5 renamed `_default_handler` → `default_handler` (cross-module import flagged SLF001).
- 1.7 had to hoist `_clean_env_before_each` autouse fixture into `tests/integration/conftest.py` after duplicating it across two integration test files (post-review patch).
- `tests/**/*.py` per-file-ignores grew over time (`S101`, then `SLF001` in 1.2, then a glob widening).

### 5. Single seam for time / sleep / random monkeypatching
1.4 codified the pattern: `monkeypatch.setattr(_transport.time, "sleep", recorder)`, not `monkeypatch.setattr("time.sleep", ...)`. 1.5 inherited it; 1.7 inherited it. **Side effect:** integration tests reach into `semvertag._transport` internals; any refactor of `_transport.py` silently breaks downstream test suites by reintroducing real sleeps. No `inject_sleep_fn` seam was added — recorded in deferred-work but not closed.

### 6. Review-finding triage discipline matured across stories
- 1.1 had no `### Review Findings` section.
- 1.2 introduced the Patches / Decision / Defer / Dismissed buckets.
- 1.5 produced 27 unique findings dedup'd to 17 patches + 8 deferred + ~22 dismissed.
- 1.6 introduced "second review pass after post-review patches" pattern (post-patch edge cases get their own deferred-work section).
- 1.7 closed Epic 1 with the same discipline: 3 decisions resolved + 6 patches + 9 deferred + 9 dismissed; sprint-status flipped `review → done` only after all patches applied and gates passed.

### 7. Spec ACs frozen at story-write time bind dev hands
- 1.4 AC1 fixed the exact 4-tuple of `RETRYABLE_EXCEPTIONS`. Review wanted `ConnectTimeout`/`PoolTimeout` added but tuple change requires AC1 revision → deferred.
- 1.6 AC2 said "field defaults preserved." Post-review patch added `Field(min_length=1)` validators → flagged as deviation from AC despite being strict-superset hardening.
- Mechanism worked, but it created friction. Possible mitigation for Epic 2: ACs phrased as "MUST include X" rather than "exact shape is Y".

## Struggles / friction points

1. **Story 1.5 review produced 17 patches in-cycle**, including a CRITICAL SSRF vulnerability on the `Link: rel="next"` follow path. The architecture sketch and dev's initial parser had 6 RFC-8288 defects. Adversarial review (`bmad-code-review`) was load-bearing for security correctness.
2. **`apply_cli_overlay` validation gap** (1.2 review): `model_copy(update=...)` skips pydantic validators. `--token raw-string` would land a bare `str` (not `SecretStr`); `--request-timeout 9999` bypassed the clamp; `--strategy garbage` accepted non-Literal values. Patched, but indicates the `model_copy` idiom was under-thought up-front.
3. **modern-di per-invocation construction fought with `@inject`** (1.7). Decorator's container-lookup contract assumed a one-shot `setup_di(...)` boundary; semvertag needs per-CLI-call container rebuild after CLI overlay. Documented as a refactor candidate in deferred-work.
4. **GitLab response-shape parsing was repeatedly under-defensive.** 1.5 review patched `get_latest_commit_on_default_branch`, `list_tags`, `check_scopes`, and `create_tag` all for missing/wrong-type guards on `response.json()`. Suggests an internal "defensive parser" helper would have saved 4 separate patches.
5. **Output instance lifecycle in `__main__.py`** (1.7). Output must exist before the try-block (to be available in `except`), but the use case wires `output=OutputsGroup.rich_output` at Factory time. Reconciled via `_run_with_output_override` + `dataclasses.replace` — works, but the override-then-swap pattern is acknowledged technical debt.

## Tech debt added by Epic 1

- **`skip_creator_parsing=True` is now the project default in `ioc.py`.** Every cross-Group dependency must be spelled out in `kwargs={...}`. modern-di's auto-wiring-by-type is effectively off. Epic 2's `ConventionalCommitsStrategy` group wiring inherits this convention.
- **Output construction is two-layered** (DI-resolved + entrypoint-local instance for `except` handlers). Future Output subtypes (e.g., structured-log format for Epic 3 doctor) inherit the override-then-swap reconciliation.
- **CLI overlay does NOT re-trigger all pydantic validators for nested CLI flags.** Story 1.2 review patched the major paths but left "CLI value survives revalidation when env conflicts" deferred. Epic 2's per-repo strategy switching via `--strategy conventional-commits` will hit this surface.
- **`Provider` and `BumpStrategy` Protocols exist but production code bypasses them.** DI Groups construct concrete classes. Future tests that want to substitute via Protocol will need DI-override scaffolding, not Protocol-based `isinstance` checks.
- **Integration tests are tightly coupled to `semvertag._transport` internals** via `monkeypatch.setattr(_transport.time, "sleep", ...)`. Any refactor of `_transport.py` silently breaks 1.5/1.7 integration suites.
- **`pyproject.toml` ships with 18 template-inherited soft issues**, none closed in Epic 1. First PyPI publish (Story 4.2) will be lumpy: sdist may ship `_autosemver_reference/`; codecov upload fails noisily on fork PRs; CI jobs have no `timeout-minutes`.
- **The `default_handler` test fixture's permissive shape** (any POST → 201; unknown path → 404) lets URL-builder typos in Epic 2/3 silently misdiagnose as `ConfigError("project not found")`.

## Forward-looking concerns for Epic 2

Epic 2 = ConventionalCommitsStrategy + per-repo switching. Story 1.7's `build_container` currently `fails-fast` for `settings.strategy != "branch-prefix"` (`semvertag/ioc.py:141-143`) — that gate is what Epic 2 unlocks.

**C1 — `Strategy` Protocol surface will need a "what do I need from the provider" capability query.**
Today the use case unconditionally calls `provider.get_default_branch()` and `get_latest_commit_on_default_branch()`. Conventional Commits scans all commits since the last tag — different read shape entirely. Either the use case grows a branch on `settings.strategy`, or `BumpStrategy` Protocol grows a capability marker. The 1.7 deferred item "`--default-branch` not consumed by `GitLabProvider.get_default_branch()`" is the same architectural seam. Risk: Epic 1 didn't carve this seam.

**C2 — CLI overlay × per-repo settings precedence is untested under stress.**
The PRD defers `.semvertag.toml` (FR23/FR24) to v1.x, so Epic 2's per-repo switching happens via env + flag only. The 1.7 deferred test gap "CLI value survives revalidation when env conflicts" is unresolved — Epic 2 will hit it the moment `--strategy conventional-commits` meets a remote `SEMVERTAG_STRATEGY=branch-prefix`.

**C3 — `BranchPrefixStrategy.decide()` first-line-only subject parsing is a precedent ConventionalCommitsStrategy can't directly inherit.**
Conventional Commits parses subject (`type(scope): description`) AND footers (`BREAKING CHANGE:`). Two different message regions. The 1.6 deferred items on CRLF `\r`, U+2028/U+2029, leading-newline messages, and `splitlines()` vs `split("\n")` all become Epic 2 blockers, not cleanup items. **Recommendation:** harden message parsing as Story 2.1's first task, not deferring again.
